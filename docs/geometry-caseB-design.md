## 1. HexOutlineBuilder API 仕様

### 1.1 状態機械

- 内部状態: 現在位置 `(x, y)`(XY 平面 = 基底面)、**方位インデックス `d ∈ {0..5}`**(実角 θ = d·60°、CCW 正)、累積ターン数 `Σturn`、直前操作 `lastOp`。
- 方位は**整数インデックスのみ**で保持し、浮動小数の角度累積を持たない(方位ドリフトが構造的に存在しない)。
- a 軸との対応: `d = 0/2/4` を a1/a2/a3(120° 間隔)と定義。反平行は `d+3`。線方位(mod 180°)は `d mod 3` が `A_AXES` の添字と一致。

### 1.2 API

```ts
class HexOutlineBuilder {
  constructor(startDir?: 0|1|2|3|4|5);                 // 既定 0
  static fromAxis(axis: 0|1|2, antiparallel?: boolean) // d = 2·axis (+3 if antiparallel)
    : HexOutlineBuilder;
  forward(len: number): this;   // 現在方位へ len 進み頂点を打つ。len > 0 を検証
  turnLeft(): this;             // d ← (d+1) mod 6(+60° 固定)
  turnRight(): this;            // d ← (d+5) mod 6(−60° 固定)
  close(): void;                // 閉路検証(§1.3)。以後の操作は不可
  points(): Array<[number, number]>; // close 後のみ。CCW 正規化済み点列
}
```

- **交互規則(構成的保証の核)**: `forward` の直後に `forward`、`turn` の直後に `turn` は **throw**。操作列は `forward (turn forward)* close` に限定される。これにより
  - 全エッジ方位 ∈ 60° 系列 — 方位が整数 d でしか表現できないため(表現不能性)。
  - 全頂点の外角 = ±60° → **内角 ∈ {120°(凸), 240°(凹)}** — ±120° 以上のターンが表現できないため。
  - 内角 180° の冗長頂点・縮退エッジ(len ≤ 0)も表現不能。
- `points()` は閉路の頂点列(最終頂点 = 始点の重複なし)。CW(Σturn = −6)で定義された場合は反転して CCW で返す(THREE.Shape の外輪郭規約に合わせる)。

### 1.3 close() の検証

操作列は forward で終わる(§1.2)ため、最後の辺から最初の辺へ戻る**継ぎ目頂点(= 始点)のターンは明示されない** — 明示ターン和は単独では ±6 にならない。close() は継ぎ目ターンを導出して検証する:

1. `lastOp === 'forward'`(turn で終わる閉路は不正)。
2. **継ぎ目ターン**: `t_close ≡ (d_start − d_end) mod 6` を導出し、**`t_close ∈ {+1(=1), −1(=5)}` を要求**(それ以外は throw)。これにより始点頂点の内角も 120°/240° に保証される(交互規則の保証が継ぎ目まで届く)。
3. **回転総和**: `Σturn_explicit + t_close = ±6`(= ±360°)を検証。それ以外は throw(±360° 超の自己周回・不足を排除)。
4. **位置閉合**: 終端位置と始点の距離 ≤ `1e-9 × max(1, Σ|len|)`。方位は整数保証だが、位置は浮動小数の辺長和なのでイプシロン検証とする(§1.4)。

任意注記(採否は CP-B2 実装者判断): 位置閉合は基底 `u = (1, 0)`・`v = (1/2, √3/2)` の係数和でも検証できる(6 方位とも u・v の整数係数結合なので、係数 2 本の float 加算に √3 が混入しない)。

### 1.4 構成的保証の範囲(明記)

| 性質 | 担保方法 |
|---|---|
| エッジ方位 60° 系列 / 内角 ∈ {120°, 240°} / 冗長頂点なし | **構成的**(API で表現不能) |
| 閉路の位置閉合 | close() の**イプシロン検証**(辺長は任意 float のため) |
| 自己交差なし(単純多角形性) | **利用側責務**(MVP では検証しない。対象 3 形態のターン列は固定であり、§4 のテストで形状ごとに担保。汎用の交差判定は将来オプション) |

### 1.5 層配置(THREE 依存の置き場所)

- ビルダー本体は **THREE 非依存**で `src/geometry/hexOutlineBuilder.ts` に新設(crystallography.ts の「純粋関数のみ」規約を保つため別ファイル。同規約の純粋クラス + 便宜関数)。
- 便宜関数(同ファイル): `dentedHexOutline(R, m, w): Array<[number, number]>` — §2 の花形断面を builder のターン列で生成し、**中心 = 原点**に正規化して返す。
- **THREE.Shape / ExtrudeGeometry 化は parts.ts 層**: 既存 `createElongatedHexPrism` 内の moveTo/lineTo/closePath イディオムを `outlineToShape(points)` / `extrudePrism(points, height)` として共通化し、既存関数もこれを使うようリファクタする(挙動不変)。監査 §2 の `toShape()` は本設計では「`points()` + parts.ts の `outlineToShape`」の 2 層に分離する — 分類層(crystallography)を THREE 非依存に保つ既定方針(growth モジュールと同じ)に従う。

## 2. 対象 3 形態の目標形状

共通: 断面アウトライン(基底面 XY)を c 軸方向に押し出して柱を作る。押し出し高さ・外接半径・レイヤー構成・配色は v1 値踏襲。

### 2.1 花形断面(辺中央凹み)— 骸晶角柱・さや・針の共通断面

正六角形(一辺 s = 外接半径 R)の**各辺の中央部を 60° 系列のコの字で深さ g だけ彫り込んだ**単一アウトライン。1 辺分のターン列(CCW、辺方位 d):

```
forward(e) → turnLeft → forward(w) → turnRight → forward(m)
→ turnRight → forward(w) → turnLeft → forward(e) → turnLeft   // 角部、次辺 d+1 へ
```

- 根拠: CCW では内部が進行方向左。右折入りの旧列は外向きタブを生成し外形が R を超えるため §3 の踏襲方針と矛盾(2026-06-11 検収で L/R 入れ替えに修正)。
- 内角: 凹み入口・出口 = **120°**(凸)/ 凹み底の両端 = **240°**(凹)/ 角部 = 120°。内角列(頂点順)= **{120, 240, 240, 120, 120}×6**。
- 辺長関係: `s = 2e + m + w`(水平投影: 側壁の cos60° = w/2 が両側)/ 凹み深さ `g = w·sin60° = (√3/2)·w`。凹み底辺(長さ m)は外周辺と平行(方位 d、内側へ g 平行移動)。
- 1 周 = 上記 × 6(ループ)。Σturn = 6 × (+1 −1 −1 +1 +1) = **+6** ✓(実装上は最後の角部 turnLeft を除いた明示和 +5 に t_close = +1 が補完される、§1.3・§4)。頂点 30・全エッジ 60° 系列・6 回対称。
- 自己交差回避の十分条件: `e > 0, m > 0, w > 0, g < apothem = (√3/2)·s`(対称性より対面・隣接凹みと干渉しない)。`dentedHexOutline` 内で検証して throw。
- 寸法目安(最終値は実装時に既存形態と実測比較して一任 — 砲弾・側面・冠柱と同方式): s = R = 0.4 / m ≈ 0.16 / w ≈ 0.08(→ g ≈ 0.069)/ e = (s−m−w)/2 ≈ 0.08。

### 2.2 骸晶角柱 (C3b)

- **置換**: 骨格 Box 6 本(0.55×1.7×0.15、+0.2 はみ出し)+ 側面凹み Box 6 枚(+ 水色エッジ線)→ **花形断面 1 本の押し出し**(R 0.4 / H 1.5)+ 全周 EdgesGeometry(COLORS.edge)。
- **踏襲**: 上下端面の凹み表現(沈めた六角柱 ×2、ハイライト色)— CylinderGeometry(…,6) は 60° 系列整合済みで監査指摘外。漏斗状の 3D 端面凹みは将来(基底面上の立体延長と同系の別設計)。
- マテリアル: 単色 COLORS.base + flatShading(現行どおり)。凹み面の個別ハイライトは廃止し、視認性は EdgesGeometry の輪郭線で確保(§3 判定観点)。
- **実装時判明(CP-B3 修正ラウンド 2、2026-06-11)**: 本節の「踏襲: 上下端面の凹み表現(沈めた六角柱 ×2)」は v1 の実構成と不一致だった。v1 の主柱は**閉キャップ**で、沈め六角柱は内部に埋没し**元々不可視** — 見かけの端面凹みは Box 骨格の +0.2 はみ出しが作る**縁の錯視**だった。よって錯視の踏襲ではなく**実ジオメトリの窪み端面**へ置換した(骸晶角柱の定義的特徴 = 中空端面の忠実化。2026-06-11 チャット判定):
  - コア: 花形断面の押し出し(高さ H − 2·lip、lip = 0.15 = v1 dentDepth 踏襲)。キャップ材 = COLORS.highlight の 2 枚組(ExtrudeGeometry グループ index 0 = キャップ / 1 = 側面)→ 窪み床がハイライト色(v1 沈め六角柱の意匠踏襲)。
  - リップリング ×2: 外形 = 同花形断面(凹み溝は全高で連続)/ 穴 = 六角形(円半径 0.28 = v1 dentRadius 0.4×0.7、頂点 = 角方向 0° + k·60°)。穴 apothem ≈ 0.243 < 凹み床中心距離 apothem − g ≈ 0.277 で干渉なし(ガードはテスト恒久化)。上下端 y ∈ ±[H/2 − lip, H/2]。
  - 外形シルエット・全高 1.5・外接半径 0.4 は不変。§3 表の「出ない差分: 上下端面の凹み」は「実窪みへの置換(ハイライト床は踏襲)」と読み替える。

### 2.3 さや (C2a)

- **置換**: 最外層(i = 0)の六角柱 + エッジ Box 6 本 → **花形断面 1 本の押し出し**(R 0.4 / H 1.6 = baseHeight·0.8、不透明)。骸晶角柱と同じ `dentedHexOutline` のパラメタ違い(薄肉・縦溝の意匠: w・m を小さめに)。
- **踏襲**: 内側の透明レイヤー 5 重(i ≥ 1、CylinderGeometry — 整合済み)。
- 将来オプション(本 CP 群では実装しない): 真の中空鞘(外六角 CCW + 内六角 CW を THREE.Shape の holes で合成)。builder は単一閉路の道具とし、リングは Shape 合成で表現する方針だけ予約。

### 2.4 針 (C1a)

- **置換(中心柱)**: i = 0 層のエッジ Box 6 本 → 花形断面の押し出し(さやと同じ扱い。R・H は現行の i = 0 層値)。i ≥ 1 の透明レイヤーは踏襲。
- **針 12 本の rotation.y — 精密化と明示式**:
  - 現状分析: 針の配置角 `φ_i = i·60° + 30°` は **mod 60° で全針共通(≡ 30°)**。CylinderGeometry(…,6) の頂点方位はローカルで 30° + k·60°(atan2(z,x) 流儀)のため、`rotation.y = 0` でも各針の頂点はたまたま radial 方向に整合している。すなわち現行は「**偶然(配置角の特殊値)に依存した暗黙整合**」であり、φ_i の定義変更で壊れる。これが監査 #4 の「ファセット方位が配置角と無関係」の実体。
  - 案 B の対応: 整合を式で明示する — `needle.rotation.y = −φ_i + Θ0`(Θ0 = CylinderGeometry の頂点位相の較正定数。実装時に頂点座標から機械検証で確定し、コメントに導出を記す)。
  - 帰結(重要): φ_i は 60° 系列 + 共通位相なので、正しい Θ0 では `−φ_i + Θ0 ≡ 0 (mod 60°)` となり、六角柱の 6 回対称により**針本体のジオメトリは v1 と完全同一**になる見込み。針の成果は「式の明示化 + vitest 恒久化」であり、3D 差分は中心柱の Box 置換のみ(§3)。
  - 結晶学注記: 全針の格子方位が mod 60° で一致することは、針束を中心柱と同一単結晶とみなす解釈とも両立する(専門家確認には追加しない)。
- needleRadius = edgeThickness/√3 ≈ 0.0866 は Box 由来の導出だった — **値は見た目踏襲のため維持**し、導出コメントを花形断面の寸法(w 等)に基づく記述へ更新。

## 3. 見た目差分の方針

Box 貼り付け → 単一アウトライン押し出しのため、**差分は必ず出る**(回帰スナップショットでの同一性主張はしない)。想定差分と判定:

| 形態 | 出る差分 | 出ない差分 |
|---|---|---|
| 骸晶角柱 | 骨格 Box の上下はみ出し(+0.2)消失 / 角部貫入消失 / 側面 Box+水色線 → 240° 凹角の彫り込み | 外接半径・全高・上下端面の凹み・配色系 |
| さや | 最外層の Box 筋 6 本 → 花形断面の縦溝 | レイヤー構成・透明度・全高 |
| 針 | 中心柱(i=0)のエッジ Box → 花形断面 | **針 12 本は形状不変**(§2.4 の回転対称) |

- 目視定番(条件出し): 骸晶角柱 = (−6, 0.10) ml66/C1f(§10 確定値の流用)。さや・針は CP-B3 で classifyOnDiagram により確定(P1d 方式): **針 (−5, 0.25) ml66/N1a / さや (−7, 0.25) ml66/N1c**。〔実装時判明: 起案時の目安 さや (−5, 0.15) は s ≈ 0.93 > C1f 上限 0.9 で ml66/N1a(針)に落ちるため −7 °C 帯へ修正。なお (−7, 0.15) は ml66/C1f(骸晶角柱)につき注意〕。手動モード(形態セレクト)での直接表示も併用可。
- 判定観点(スクショ): ① 凹みが各辺中央・6 回対称 ② 貫入・はみ出しの消失 ③ 凹角エッジが 60° 系列に見える(水平断面で) ④ v1 のファミリー感(シルエット・配色)維持 ⑤ 針: 中心柱以外不変。

## 4. テスト計画(vitest)

役割分担 — 構成的保証(§1.4 表の 1 行目)はテスト不要が原則だが、**生成物の実測検算を回帰二重化として最小限残す**(crystallography.test.ts の interiorAngle / edgeAnglesFromAxis 流儀を再利用):

1. builder 不変条件(機械検証): 非閉路で throw / `t_close ∉ {+1, −1}` で throw / `Σturn_explicit + t_close ≠ ±6` で throw / 連続 forward・連続 turn で throw / len ≤ 0 throw / CW 定義 → CCW 正規化。
2. 正六角形ゴールデン: `fromAxis(0) → forward(s) → (turnLeft → forward(s))×5 → close()`(明示和 +5、t_close = +1)の points が **CylinderGeometry(…,6) の断面頂点と一致**(中心正規化 + 位相合わせのスナップショット比較 — 既存 6 形態との互換の証明)。
3. 花形断面(dentedHexOutline): **辺ユニット×6 から最後の角部 turnLeft を除いて close**(t_close = +1 が補完)。内角列が **{120,240,240,120,120}×6** / 全辺方位 60° 系列 / 60° 回転で自己一致(6 回対称)/ 自己交差条件違反(g ≥ apothem 等)で throw。
4. 形態検証: 骸晶角柱・さや・針(中心柱)に BoxGeometry が存在しない / 押し出しメッシュの頂点が断面アウトラインと整合 / 針: 全 12 本のワールド頂点方位 − φ_i ≡ 一定 (mod 60°) / dispose 正常 / 同一 seed → 同一形状(針)。
5. 既存 57 件回帰維持(growth・冠柱・砲弾・側面・条件図を含む)。

想定件数: builder 系 9–11 + 形態系 5–7 = **+14〜18(57 → 71〜75)**(設計段階の目安。確定は各 CP 報告で)。
